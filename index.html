Cosmos — Peer Learning Platform

Founder & Developer · Aug 2024 – Present · React, Firebase, Real-time Sync

The Problem: Every year, thousands of CS students in India grind 8-10 hours daily—solving DSA problems, building projects, preparing for interviews. Most don't get the internship. Not because they lack effort, but because effort without feedback is noise.

Students don't know which hours actually moved the needle, whether their "progress" is real or performative, or how their preparation compares to peers who succeeded. Traditional solutions fail: mentors don't scale, communities become noise, accountability apps track tasks—not outcomes.

The Thesis: Evidence-Based Design

I spent weeks reviewing 15+ peer-reviewed papers on learning science. The evidence pointed to specific mechanisms:

  • Retrieval practice (d = 0.70): Daily proof submissions, not just check-ins
  • Implementation intentions (d = 0.65): Goal-setting tied to specific actions
  • Cooperative learning (d = 0.40): Pod-based peer feedback loops
  • Public commitment: Weekly reflections shared with pod

Most "accountability" apps ignore this research. They gamify streaks without understanding why streaks sometimes work—and when they backfire.

The Product

Cosmos organizes students into pods—micro-communities of 5-8 people with similar goals (e.g., "SDE internship at product companies, Summer 2026"). Each pod runs on a weekly rhythm:

  • Daily proof submissions: Not "I studied 4 hours" but "Here's the problem I solved, here's where I got stuck"
  • Peer feedback: Pod members review each other's work, catch blind spots
  • Streak tracking: Consistency signals, not vanity metrics
  • Weekly reflections: What worked? What didn't? Shared transparently

No instructors. No courses. Just structured peer pressure from people who understand your exact situation.

Technical Implementation

  • Frontend: React with Vite for fast iteration
  • Backend: Firebase (Firestore for real-time sync, Auth for user management, Hosting)
  • Key systems: Real-time pod activity feeds, streak calculation with timezone handling, proper session management
  • Security incident & recovery: Accidentally exposed API keys in public commit. Detected within hours, rotated credentials, rebuilt deployment pipeline with proper key restrictions via Google Cloud Console. Documented incident and prevention measures.

Current State & Honest Challenges

30 beta pods—but facing the classic cold start problem. Empty pods kill retention: student joins, sees no activity, leaves. Death spiral.

What I'm iterating on: consolidating to 1-3 genuinely active pods before scaling, community-led growth via existing WhatsApp/Telegram groups, activation mechanics—what gets a new member to post within 24 hours?

This is the hardest part of building a community product. No amount of features fixes an empty room.

What's Next

AI-powered journal analysis (planned): Integrating Gemini API to surface patterns—"You've spent 60% of time on easy problems" or "Your stuck points cluster around dynamic programming." The goal: make effort legible.

FinSight — Market Anomaly Detection for Retail Traders

Python, FastAPI, PostgreSQL, JWT Auth

The Problem: Retail traders compete against hedge funds with sophisticated tools that detect market anomalies before they become obvious. These institutional tools cost thousands per month and require analyst teams. The average person has no access to this intelligence.

What FinSight Does: AI-powered platform bringing institutional-grade anomaly detection to retail traders—watching markets across multiple timeframes and alerting when something unusual happens.

How It Works

  • Multi-Timeframe Anomaly Detection: Analyzes patterns across daily, weekly, and monthly horizons simultaneously—a stock might look normal daily but show warning signs weekly
  • Causal Learning Algorithms: Understands why anomalies occur, not just correlations—reducing false positives
  • Agent-Based Decision System: Multiple specialized "agents" focus on different market behaviors; collective judgment outperforms any single approach
  • Composite Confidence Scoring: Every signal includes strength evidence—helping prioritize which anomalies deserve attention

Technical Implementation

  • Backend: FastAPI (Python) with async architecture for concurrent requests
  • Database: PostgreSQL with asyncpg—stores accounts, watchlists, signal history
  • Auth: JWT-based token system for secure sessions
  • API: RESTful routes for auth, signals, watchlist management, user profiles
  • Frontend: Single HTML file with Tailwind CSS—no build process, deploys anywhere

The Journey

Started as a CLI tool with hardcoded user ID. Rebuilt from ground up into multi-user web platform: fixed SQL injection vulnerabilities, built user management, created HTTP API layer, designed zero-dependency frontend.

Status: Functional API, user auth, watchlist management, signal delivery. Ready for cloud deployment (Railway + Vercel + Supabase on free tiers).
